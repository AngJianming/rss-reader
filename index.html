    Rust Language Essentials: Variables, Constants, and Mutability Explained for Beginners
    Continuing to our last post on the rust language fundamentals, today we will discuss on language foundations. What are variables, constants and Mutability works in Rust. Understanding how to store and manage data is fundamental. Rust has specific rules around the variables and mutability that are key to its safety guarantees. let : Variables are declared using the let keyword: let variable_name = value; Example: let my_var = 5; By default, variables in Rust are immutable. Once a value is assigned with let , it can not be changed. This default behaviour helps prevent the accidental data modification and makes code easier to reason about. mut : To allow a variable'd value to change, you must explicitly use the mut keyword: let mut variable_name = value; Example: let mut my_var = 123; my_var = 345; const : Declared using the const keyword: const NAME: Type = value; Always immutable, can not use mut with constants. Must have their type annotated Can only be set to a constant expression which is computable at compile time. Often used for hardcoded values and conventionally named in UPPER_SNAKE_CASE. Declaring a new variable with the same name as previous one using let. Example: let x = 5; let x = x + 1; The second x is a new variable shadowing the first. Key difference from mut: Shadowing creates a new variable, allowing you to change the type of the bound to the name. mut changes the value in place but not the type. Useful for transforming a value while reusing the variable name. Variables in Rust store data, which can be of various basic data types like integers, floats, booleans, and characters ( we will cover this in more details in coming articles).
