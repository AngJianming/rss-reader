In other words, inner functions can "close over" variables defined in an outer function ‚Äî keeping them alive between calls. Even after createFibonacci() has returned, the fib() function still remembers the memo object. This is the closure in action: the inner function keeps access to its outer scope. Closures allow us to: Preserve state across function calls without polluting the global scope. Create factory functions like createFibonacci() that generate self-contained logic.
